一、怎样编辑 HTML 比较方便

1.1 编辑器选择

编辑器适用场景核心优势
VS Code日常开发、项目级工作Emmet 内置、插件生态最强、Git 集成
Sublime Text快速打开大文件启动极速、资源占用低、多光标编辑
Cursor / WindsurfAI 辅助编程原生 AI 内联、代码生成、聊天式编辑
CodePen / JSFiddle快速原型验证实时预览、分享链接、零配置

1.2 必装插件与配置（VS Code）

  • Emmet：已内置。输入 div.container>ul>li*5 按 Tab 即可展开完整结构。
  • Auto Rename Tag：修改开始标签时自动同步结束标签。
  • Live Server：本地实时刷新浏览器，保存即刷新。
  • Prettier：统一格式化，避免缩进混乱。
  • IntelliSense for CSS class names：自动提示已定义的 class。
💡 效率技巧：VS Code 按 Ctrl + D 选中下一个相同词，Alt + 点击 多光标同时编辑，Ctrl + Shift + L 选中所有匹配项。

1.3 Emmet 速查（节省时间 50% 以上）

/* 基础结构 */
header#top>nav>ul>li*4>a{链接 $}

/* 展开为 */
<header id="top">
  <nav>
    <ul>
      <li><a href="">链接 1</a></li>
      <li><a href="">链接 2</a></li>
      <li><a href="">链接 3</a></li>
      <li><a href="">链接 4</a></li>
    </ul>
  </nav>
</header>

/* 带属性的图片网格 */
section.gallery>img[src="img/$.jpg" alt="图片 $"]*9

/*  siblings 与分组 */
(main>article*2>h2+p)+(aside>div.widget*3)

1.4 浏览器开发者工具

按 F12 打开 DevTools，最实用的面板：

  • Elements：实时编辑 HTML/CSS，所见即所得，改完再复制回代码。
  • Console：测试 JavaScript 片段，$0 引用当前选中元素。
  • Network：查看 CSS/JS 文件是否加载、体积大小、缓存状态。
  • Device Toolbar（Ctrl+Shift+M）：模拟手机、平板响应式布局。

二、怎样提示 AI 修改 HTML 最省资源

2.1 核心原则：上下文最小化

AI 按 token 计费，发送冗余内容既浪费钱又降低精度。只发送需要改的部分，而非整个文件。

2.2 高效 Prompt 结构（推荐模板）

角色设定 + 上下文 + 具体指令 + 输出格式

你是一个前端开发专家。请修改以下 HTML 片段：
【粘贴 20-50 行需要修改的代码】

要求：
1. 将卡片布局改为 CSS Grid，3 列响应式；
2. 图片添加懒加载属性；
3. 不要改动任何 class 名称；
4. 只返回修改后的 HTML 代码块，无需解释。

2.3 节省 Token 的具体技巧

低效做法高效做法节省估算
发送整个 500 行 HTML 文件 只发送相关 30 行，用注释标记修改位置 ~80%
"帮我看看这个网页有什么问题" "以下导航栏在 Safari 下错位，只返回修正后的 CSS" ~60%
让 AI 从零写完整页面 提供组件模板，让 AI 填充内容/变体 ~70%
重复粘贴相同的 CSS 变量 "使用项目已有的 :root 变量，不重新定义" ~30%

2.4 标记法：精确定位修改点

在代码中用特殊注释包裹需要 AI 处理的区域：

<!-- AI-MODIFY-START -->
<div class="old-card">
  <h2>标题</h2>
  <p>描述文字</p>
</div>
<!-- AI-MODIFY-END -->

<!-- 提示词：请将 AI-MODIFY 区域内的代码转换为 article 语义标签，
     并添加 schema.org 的 Article 结构化数据，其余部分保持原样。 -->
⚠️ 避免陷阱：不要一次给 AI 太多任务（超过 3 个独立修改点）。分批处理，每轮只聚焦一个目标，出错率显著降低。

2.5 让 AI " diff 模式 "输出

如果只需要小改动，要求 AI 以 diff 格式返回，再手动应用：

请以下列格式输出修改：
- 删除的行前缀：-
+ 新增的行前缀：+
未改行不输出

三、CSS 文件管理与内联样式处理

3.1 CSS 文件组织策略

方案 A：单文件小型项目（推荐 < 500 行）

styles.css          /* 唯一样式表 */
  ├── CSS Variables (:root)
  ├── Reset / Normalize
  ├── Utility classes
  ├── Layout (header, footer, sidebar)
  ├── Components (buttons, cards, forms)
  └── Media Queries (放在底部或紧跟组件)

方案 B：多文件中型项目

css/
  ├── base/
  │   ├── reset.css       /* 清除默认样式 */
  │   ├── variables.css   /* 颜色、间距、字体变量 */
  │   └── typography.css  /* 标题、段落、链接 */
  ├── layout/
  │   ├── grid.css        /* 网格系统 */
  │   ├── header.css
  │   └── footer.css
  ├── components/
  │   ├── buttons.css
  │   ├── cards.css
  │   └── forms.css
  ├── pages/
  │   ├── home.css        /* 仅首页需要的特殊样式 */
  │   └── article.css
  └── main.css            /* @import 或构建工具合并 */

方案 C：WordPress / 无构建工具场景

assets/
  ├── css/
  │   ├── global.css      /* 全站通用：排版、工具类 */
  │   ├── header.css      /* 导航、页头 */
  │   ├── content.css     /* 文章正文样式 */
  │   └── components.css  /* 按钮、卡片等复用组件 */
  └── js/
      └── main.js

3.2 CSS 变量系统（设计令牌）

:root {
  /* 颜色 */
  --color-primary: #2563eb;
  --color-primary-dark: #1d4ed8;
  --color-text: #1e293b;
  --color-text-light: #64748b;
  --color-bg: #f8fafc;
  --color-surface: #ffffff;
  --color-border: #e2e8f0;

  /* 间距 */
  --space-xs: 0.25rem;   /* 4px  */
  --space-sm: 0.5rem;    /* 8px  */
  --space-md: 1rem;      /* 16px */
  --space-lg: 1.5rem;    /* 24px */
  --space-xl: 2rem;      /* 32px */

  /* 字体 */
  --font-sans: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
  --font-mono: "SF Mono", Monaco, Consolas, monospace;

  /* 圆角与阴影 */
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 12px;
  --shadow-sm: 0 1px 2px rgba(0,0,0,0.05);
  --shadow-md: 0 4px 6px rgba(0,0,0,0.07);
}
✅ 好处：AI 修改时只需引用变量名，不会出现"把蓝色改成另一种蓝色"的歧义。提示词里写"使用 --color-primary"比描述颜色精确得多。

3.3 内联样式（style 属性）处理策略

场景处理方式示例
完全静态样式 必须移入 CSS 文件 <div style="color:red"> → .alert { color: red; }
动态计算值（JS） 用 JS 修改 style 属性 el.style.width = computedValue + 'px'
仅个别元素微调 使用 Utility class <div class="mt-0">（margin-top: 0）
第三方嵌入代码 保留但用 !important 覆盖 .my-widget { width: 100% !important; }
🚫 绝对避免：内联样式难以被 AI 批量修改、无法响应媒体查询、增加 HTML 体积、不利于缓存。WordPress 古腾堡块常生成内联样式，建议通过主题 CSS 覆盖。

3.4 Utility Class 设计（不依赖框架）

/* 小型项目自己维护一套 utility，比引入 Tailwind 更轻 */
.mt-0 { margin-top: 0; }
.mt-1 { margin-top: var(--space-sm); }
.mt-2 { margin-top: var(--space-md); }
.text-center { text-align: center; }
.flex { display: flex; }
.gap-2 { gap: var(--space-md); }
.rounded { border-radius: var(--radius-md); }

四、熟练掌握原生三件套需要什么（内容层面）

4.1 HTML：语义与结构

必须掌握：
  • 语义标签：article, section, nav, aside, header, footer, main
  • 表单元素与属性：type, required, pattern, label 关联
  • 可访问性：alt, aria-label, aria-expanded, role
  • 元数据：meta viewport, description, Open Graph
  • 结构化数据：JSON-LD / schema.org
进阶：
  • 模板元素：template, slot, Shadow DOM 基础
  • 懒加载：loading="lazy", decoding="async"
  • 响应式图片：srcset, sizes, picture 元素
  • 自定义数据属性：data-* 与 JS 交互

4.2 CSS：布局与视觉

必须掌握：
  • 现代布局：Flexbox 和 CSS Grid
  • 响应式：媒体查询 @media，移动优先写法
  • 定位：static, relative, absolute, fixed, sticky
  • 层叠与优先级：特异性计算、!important 慎用原则
  • 盒模型：content-box vs border-box
  • 变量与计算：var(), calc(), clamp()
进阶：
  • 动画：transition, @keyframes, 性能优化（transform/opacity）
  • 现代选择器：:is(), :where(), :has()
  • 容器查询：@container（组件级响应式）
  • 逻辑属性：margin-inline, padding-block
  • 混合模式与滤镜：mix-blend-mode, backdrop-filter

4.3 JavaScript：交互与逻辑

必须掌握：
  • DOM 操作：querySelector, addEventListener, createElement
  • 事件机制：冒泡、捕获、委托
  • 异步：Promise, async/await, fetch API
  • ES6+：解构、展开、模板字符串、箭头函数、模块化
  • 数据类型与转换：深拷贝、JSON 序列化
  • 调试：console 全家桶、断点、错误处理 try/catch
进阶：
  • Web API：Intersection Observer, Resize Observer
  • 存储：localStorage, sessionStorage, IndexedDB
  • 性能：防抖 debounce、节流 throttle、requestAnimationFrame
  • 模块化：ES Modules, 动态 import()
  • 安全：XSS 防范、CSRF Token、内容转义

4.4 知识检验清单

□ 能不看文档写出三栏布局（Grid + Flex）
□ 能解释 "盒模型" 并用 border-box 修复布局问题
□ 能手写防抖函数并说明适用场景
□ 能正确使用事件委托处理动态生成的列表
□ 能写出符合 WCAG 2.1 AA 标准的表单
□ 能用 Fetch 完成 GET/POST 并处理错误边界
□ 能诊断并修复 specificity 导致的样式覆盖失败

五、在 WordPress 中插入代码片段

5.1 插入位置决策树

需要添加代码？
  ├── 仅特定页面/文章？
  │     └── 使用「自定义 HTML 块」或「短代码」
  ├── 全站生效的 CSS？
  │     └── 外观 → 自定义 → 额外 CSS
  ├── 全站生效的 JS / <head> 代码？
  │     └── 子主题 functions.php 或专用插件
  └── 复杂的交互组件？
        └── 自定义短代码 + 外部 CSS/JS 文件

5.2 方法 1：主题定制器（额外 CSS）

路径：WordPress 后台 → 外观 → 自定义 → 额外 CSS

✅ 适合：快速修复、小范围样式调整、不修改文件。
❌ 缺点：代码分散、无法版本控制、换主题会丢失。

5.3 方法 2：子主题（推荐长期维护）

/* 1. 在 wp-content/themes/ 创建子主题文件夹，例如 my-child/ */

/* 2. 创建 style.css，头部必须包含： */
/*
Theme Name:   My Child Theme
Template:     twentytwentyfour    ← 父主题文件夹名称
Version:      1.0
*/

/* 3. 创建 functions.php 引入父主题样式和自定义 JS */
<?php
add_action('wp_enqueue_scripts', function () {
    // 继承父主题样式
    wp_enqueue_style('parent-style', get_template_directory_uri() . '/style.css');
    
    // 添加自定义全局样式
    wp_enqueue_style('custom-style', get_stylesheet_directory_uri() . '/css/custom.css', [], '1.0');
    
    // 添加自定义 JS（footer 加载）
    wp_enqueue_script('custom-script', get_stylesheet_directory_uri() . '/js/custom.js', [], '1.0', true);
});
?>

5.4 方法 3：使用插件管理代码

插件用途适用场景
Insert Headers and Footers在 <head> 或 </body> 前插入代码统计代码、第三方脚本
Code Snippets安全添加 PHP 片段替代 functions.php，防止主题更新丢失
WPCode条件逻辑插入 CSS/JS/PHP只在特定页面加载特定代码
Custom CSS-JS-PHP按页面管理代码页面级微调

5.5 方法 4：自定义短代码（Shortcode）

在子主题的 functions.php 或 Code Snippets 插件中添加：

<?php
// 注册短代码 [info_box title="标题" color="blue"]内容[/info_box]
add_shortcode('info_box', function ($atts, $content = null) {
    $atts = shortcode_atts([
        'title' => '提示',
        'color' => 'blue',
    ], $atts, 'info_box');
    
    $colors = [
        'blue'   => ['bg' => '#eff6ff', 'border' => '#2563eb', 'text' => '#1e40af'],
        'green'  => ['bg' => '#ecfdf5', 'border' => '#059669', 'text' => '#065f46'],
        'orange' => ['bg' => '#fffbeb', 'border' => '#d97706', 'text' => '#92400e'],
    ];
    $c = $colors[$atts['color']] ?? $colors['blue'];
    
    return '<div style="background:' . esc_attr($c['bg']) . ';' .
           'border-left:4px solid ' . esc_attr($c['border']) . ';' .
           'padding:16px 20px;margin:16px 0;border-radius:0 8px 8px 0;">' .
           '<strong style="color:' . esc_attr($c['text']) . ';display:block;margin-bottom:6px;">' .
           esc_html($atts['title']) . '</strong>' .
           '<div style="color:' . esc_attr($c['text']) . ';">' . do_shortcode($content) . '</div>' .
           '</div>';
});
?>

然后在古腾堡编辑器中使用「短代码」块输入：[info_box title="注意" color="orange"]这里是警告内容[/info_box]

5.6 WordPress 中 AI 修改的最佳实践

  1. 先在本地/ staging 环境测试：用 LocalWP、DevKinsta 或 Docker 搭建本地 WordPress。
  2. 导出代码片段：从「额外 CSS」或插件中复制目标代码，粘贴给 AI，而非发送整个主题文件。
  3. 要求 WordPress 安全输出：提示词中强调使用 esc_html(), esc_attr(), wp_kses_post() 防止 XSS。
  4. 版本控制：即使是「额外 CSS」，也建议在本地备份到 wordpress-css-backup.css，方便 AI 对比修改。

六、AI 辅助修改内容代码的工作流

6.1 推荐工作流（逐轮迭代）

第 1 轮：需求描述
    ↓ 你 → AI
    "我需要把以下博客文章中的引用块改成带左侧彩色边框的提示框样式"

第 2 轮：提供上下文（精简）
    ↓ 你 → AI
    【粘贴文章 HTML 中的 3-5 个 blockquote 示例，共约 40 行】
    "项目使用 CSS 变量，主色调 --color-primary: #2563eb"

第 3 轮：获取方案
    ↓ AI → 你
    【AI 返回 HTML + CSS 代码块】

第 4 轮：本地验证
    ↓ 你
    在 CodePen / 本地文件中粘贴测试，截图或描述问题

第 5 轮：精准修正
    ↓ 你 → AI
    "很好，但移动端下左边框太宽。请把 border-left 在 @media (max-width:640px) 下改为 3px，其余不变。"

第 6 轮：应用与备份
    ↓ 你
    将最终代码写入 WordPress 子主题或「额外 CSS」，同时保存到本地备份文件

6.2 分场景 Prompt 模板

场景 A：修改现有组件样式

【角色】资深前端工程师
【上下文】以下是我项目中的按钮组件 CSS，使用 CSS 变量：
[粘贴 CSS，约 20 行]
【任务】添加一个 .btn-ghost 变体：透明背景、带 1px 边框、hover 时背景填充为主色。
【约束】复用已有变量，不要新增颜色值；输出完整的 .btn-ghost 规则即可。

场景 B：HTML 结构重构

【角色】语义化 HTML 专家
【上下文】以下是我 WordPress 文章中的纯 div 结构：
[粘贴 HTML]
【任务】将所有 div 替换为合适的语义标签（article, section, time, figure 等），保留所有 class 和 id。
【约束】确保 schema.org 的 Article 结构化数据可以正确附着；只输出修改后的 HTML。

场景 C：调试样式问题

【角色】CSS 调试专家
【上下文】在 iOS Safari 下，以下导航栏出现横向滚动条：
[粘贴相关 HTML + CSS，约 30 行]
【任务】找出导致溢出的原因并给出修复代码。
【约束】不要改变桌面端布局；优先使用 CSS 方案，不用 JS。

6.3 与 AI 协作的代码审查清单

  • 检查特异性：AI 是否随意使用了 !important 或过度嵌套的选择器？
  • 检查可访问性：AI 生成的交互组件是否有 keyboard 支持、ARIA 属性？
  • 检查性能：是否有不必要的重绘属性（如频繁修改 width/height/top/left）？
  • 检查响应式：AI 是否只在桌面端测试？要求补充移动端规则。
  • 检查安全性：WordPress 代码中是否有未转义的输出？

6.4 建立个人代码片段库

用 AI 生成常用组件后，保存到本地或 GitHub Gist，形成可复用资产：

snippets/

  │   ├── responsive-table.html
  │   ├── image-lightbox.html
  │   └── breadcrumb-schema.html
  ├── css/
  │   ├── flex-center.css
  │   ├── card-grid.css
  │   ├── underline-animation.css
  │   └── dark-mode-toggle.css
  └── js/
      ├── mobile-nav-toggle.js
      ├── scroll-reveal.js
      └── lazy-load-images.js

下次需要时，直接复制基础版本给 AI："基于我的卡片网格模板，改为 4 列并添加悬停放大效果"。

🎯 最终建议：把 AI 当作"超级自动补全 + 代码审查员"，而非替代你思考的魔法工具。你负责定义问题、验证结果、理解原理；AI 负责生成草稿、提供变体、解释语法。这样的分工最高效，也最利于你持续精进原生三件套的能力。

本指南以原生 HTML/CSS/JS 为核心，适用于所有无框架或 WordPress 环境。
