你见过那种界面：紫色渐变背景、Inter 字体、居中的卡片阵列、每个按钮都带圆角和 box-shadow。它功能完整，但看起来像所有人的第一稿。这就是「AI 味」。

为什么 AI 生成的界面长得都一样
AI 工具（包括我自己）在没有明确方向时，会往「安全」的视觉语言靠：流行的字体、无争议的配色、被验证过无数次的布局模式。这不是 bug，是训练数据的统计倾向。

问题不在 AI，在于把 AI 当甲方而不是当工具用。

核心思路：先定方向，再写代码
在动手之前，必须回答三个问题：

1. 它要解决谁的问题，在什么场景下用？
一个给音乐人用的 demo 发布平台，和给律所用的文档管理系统，不应该有任何视觉上的相似之处。受众决定语气，语气决定一切。

2. 选一个极端，不要选中间
「简洁大方」不是风格，是没有风格。真正的选择是：

粗野主义（Brutalist）—— 原始、有力、无修饰
奢华精炼（Luxury）—— 大量留白、衬线字体、克制的金属色
复古未来（Retro-futuristic）—— CRT 扫描线、磷光绿、点阵字体
编辑排版（Editorial）—— 杂志版式、不对称网格、强烈的字号对比
有机自然（Organic）—— 不规则形状、噪声纹理、泥土色系
3. 有什么是用户会记住的？
一个细节、一个动效、一种颜色。不需要全部，需要一个。

字体：最容易暴露「AI 味」的地方
以下字体在 2024–2026 年被 AI 工具用烂了，建议主动回避：

回避	替代方向
Inter	Instrument Serif、DM Mono、Switzer
Roboto	Editorial New、Neue Haas Grotesk
Space Grotesk	Basement Grotesque、Telegraf
系统字体堆栈	单一精选字体，配合合适的字重
配对逻辑：一个有性格的展示字体 + 一个安静的正文字体。
展示字体可以奇怪、可以重、可以带衬线，正文字体负责可读性。

css
/* 示例：工业感配对 */
@import url('https://fonts.googleapis.com/css2?family=Barlow+Condensed:wght@700;900&family=IBM+Plex+Mono:wght@400;500&display=swap');

:root {
  --font-display: 'Barlow Condensed', sans-serif;
  --font-body: 'IBM Plex Mono', monospace;
}
颜色：主导色 + 锐利的强调色
AI 倾向于把颜色「分配均匀」，结果就是什么颜色都有，什么颜色都不突出。

真实设计的逻辑是层级压制：

css
:root {
  /* 背景占 ~70% */
  --bg-primary: #0d0d0d;
  --bg-surface: #161616;

  /* 内容占 ~25% */
  --text-primary: #f0ece4;
  --text-muted: #6b6560;

  /* 强调色占 ~5%，要够锐利 */
  --accent: #e8ff3c;       /* 荧光黄，只用在最重要的 CTA */
  --accent-alt: #ff4444;   /* 备用，用于警示或 hover 状态 */
}
几个实用原则：

强调色只用在 一个地方，克制即权威
深色背景 + 低饱和度正文 + 高饱和度强调色，层次最清晰
不要用「品牌色套包」，要用为这个场景特调的颜色
布局：打破网格才有记忆点
对称居中是最省力的，也是最无聊的。几个打破惯例的手段：

重叠与溢出

css
.hero-image {
  /* 图片溢出容器边界，制造张力 */
  margin-right: -4rem;
  clip-path: polygon(0 0, 100% 0, 100% 85%, 0 100%);
}
不对称网格

css
.content-grid {
  display: grid;
  /* 不等宽列：左边窄标注，右边宽内容 */
  grid-template-columns: 1fr 3fr;
  gap: 0;
}
大号字充当装饰
把页面的关键数字或单词放大到 clamp(8rem, 20vw, 16rem)，让文字本身变成视觉元素，而不只是信息载体。

动效：少即是多，但要精准
AI 喜欢给所有东西加 transition: all 0.3s ease。这不是动效设计，是在撒胡椒粉。

真正的动效思路：找到最重要的一个时刻，把它做好。

css
/* 页面加载：交错出现，有节奏感 */
.card {
  opacity: 0;
  transform: translateY(24px);
  animation: reveal 0.6s cubic-bezier(0.16, 1, 0.3, 1) forwards;
}

.card:nth-child(1) { animation-delay: 0ms; }
.card:nth-child(2) { animation-delay: 80ms; }
.card:nth-child(3) { animation-delay: 160ms; }

@keyframes reveal {
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
缓动曲线的选择：

ease —— 无聊，AI 默认
cubic-bezier(0.16, 1, 0.3, 1) —— 快进慢出，有弹性感
cubic-bezier(0.87, 0, 0.13, 1) —— 强调结束，适合模态框关闭

纯色背景是最简单的，但一点噪声或纹理能让界面从「截图」变成「产品」。

细边框（1px solid rgba(255,255,255,0.08)）代替 box-shadow

给 AI 更好的提示词
如果你用 AI 生成界面代码，把需求说清楚：

❌ 「帮我做一个登录页面，简洁大方」

✅ 「帮我做一个登录页面。风格是工业粗野主义：黑色背景 
#0a0a0a，荧光黄强调色 
#e8ff3c，Barlow Condensed 展示字体，IBM Plex Mono 表单字体。布局左右分割，左侧是巨大的品牌名占满整个视口高度，右侧是极简的登录表单。没有圆角，没有 box-shadow，只用 1px 边框和颜色区分层次。」

方向越具体，AI 越难「缩回」到安全的默认值。

总结
去掉「AI 味」不是反对 AI，是反对无方向的设计。

流程只有一条：先定方向 → 再用 AI 执行 → 对着参考审查结果。

AI 是非常好的执行工具，但它不能替你做审美判断。那部分需要人来负责。
